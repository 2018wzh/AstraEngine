# Editor Production Checklist

Status: Goal-aligned creator workflow audit ledger / Editor not implemented
Last audited: 2026-06-19

Purpose: track whether AstraEditor reaches the UE-level creator workflow required by AstraEngine: a real desktop Qt editor that can create, edit, debug, package, validate, and release 2D/VN projects without giving the packaged runtime an Editor dependency.

Legend:

- [x] Code-backed complete in the current source tree, with automated test, CLI, or committed report evidence.
- [~] Partial reusable runtime foundation or design contract exists, but no Editor-specific implementation.
- [ ] Planned or not implemented.

Rule: for Editor-specific features, `Code: none` means no `Engine/Editor` target or public Editor API exists yet. Reusable runtime foundations are listed as prerequisites, not as Editor completion. The row format is:

`- [status] ID - Requirement. Goal: <design goal>. Supports: <supporting elements>. Gate: <acceptance gate>. Design: <docs>. Code: <files/targets or none>. Evidence: <tests/CLI/reports or none>. Gap: <remaining production work>.`

## Goal And Boundary Gates

Supports: dockable workbench, project/session model, commands, panels, runtime embedding. Gate: `Engine/Editor` builds on Windows/Linux/macOS and opens a project. Design: ADR 0002, `editor-and-pipeline.md`, `editor-ui-ai-collaboration-prototype.md`. Code: none; only `ASTRA_BUILD_EDITOR` option exists. Evidence: none. Gap: create `Engine/Editor` target, Qt shell, startup flow, platform builds.
- [ ] ED-GOAL-002 - A new creator completes Template -> Project -> Content -> Script/Lua/Graph/Timeline -> PIE -> Cook -> Package -> Release Gate without manual file edits. Goal: creator-facing production workflow. Supports: Project Wizard, Content Browser, editors, Inspector, PIE, package panel. Gate: CreatorWorkflow sample passes tutorial automation. Design: `goals.md`, `samples-and-test-matrix.md`, `editor-runtime-creator-contract.md`. Code: none. Evidence: none. Gap: entire Editor workflow.
- [ ] ED-GOAL-003 - Editor uses public runtime DTO/API only and never becomes a packaged runtime dependency. Goal: runtime/editor separation. Supports: EditorRuntimeSession, InspectRequest, DebugCommand, SourcePatchProposal, release reports. Gate: CreatorWorkflow package launches after Editor closes and package manifest contains no Editor module. Design: `architecture.md`, `editor-runtime-creator-contract.md`. Code: none. Evidence: runtime PackageLaunch proves runtime side only. Gap: Editor integration and dependency regression tests.
- [~] ED-GOAL-004 - Runtime foundations needed by Editor already exist for many panels but are not Editor features yet. Goal: reuse EngineCore rather than duplicate systems. Supports: AssetRegistry, PropertySystem, RuntimeWorld, SaveV2, ReplayStream, Media reports, Script debug DTOs. Gate: Editor panels bind these public DTOs. Design: `implementation-coverage.md`. Code: `Engine/Runtime/*`, `Engine/Programs/astra`. Evidence: runtime CTest/CLI evidence only. Gap: Editor session API and UI bindings.
