# AstraEmu Production Checklist

Status: Goal-aligned standalone compatibility toolkit audit ledger / AstraEmu not implemented
Last audited: 2026-06-19

Purpose: track whether AstraEmu can run original-release legacy VN data as a standalone toolkit without converting it into Astra canonical source, modifying foreign source, or becoming a prerequisite for NativeVN authoring. The first production acceptance target is 100% full-content-flow coverage for three local commercial cases:

- Artemis: `Sakura no Uta 10th Anniversary Edition`
- KrKr / KAG / TJS / XP3: `Senren Banka`
- BGI / Ethornell: `Subarashiki Hibi 15th Anniversary Edition`

Legend:

- [x] Code-backed complete in the current source tree, with automated test, CLI, or committed report evidence.
- [~] Partial reusable AstraEngine foundation or design contract exists, but no AstraEmu-specific implementation.
- [ ] Planned or not implemented.

Rule: AstraEmu completion requires checked-in implementation plus committed `astra.emu.local_case_report.v1` reports. Commercial source files, unpacked payloads, and private absolute paths must not be committed. The row format is:

`- [status] ID - Requirement. Goal: <design goal>. Supports: <supporting elements>. Gate: <acceptance gate>. Design: <docs>. Code: <files/targets or none>. Evidence: <tests/CLI/reports or none>. Gap: <remaining production work>.`
