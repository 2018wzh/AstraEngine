# 视觉规范

## 方向

画面围绕“玻璃雨中的信号”建立冷静、低饱和的夜间视觉。玻璃、雨痕、反射和信号波形是可复用的形状语言；它们用于层次、转场和状态提示，不承担未定义的剧情信息。

## 版式约束

- 关键内容区域保持稳定 safe area，文本和交互控件不得贴近裁切边界。
- 场景层、角色层、前景遮罩和系统 UI 保持独立层级；任何层级变化必须能在 manifest 中绑定稳定 id。
- 高对比信息使用 UI token，不从具体素材颜色临时派生。
- 视觉 reference、checkpoint 和对比 artifact 只保留脱敏 hash、尺寸、区域 id 和诊断，不把截图或中间位图写入本目录。

## 颜色与材质

主色使用深蓝灰背景、冷白文本和少量信号青；警示状态使用受控的琥珀色。玻璃高光应保持低面积、低亮度，避免遮盖文字和可交互区域。雨痕可以重复，但同一 scene 中不得用重复纹理制造可误解的角色或路线提示。

## 动效边界

后续实现需要为镜头、转场、波形和雨幕分别声明时长、缓动和取消点。当前文档不声明任何已实现的 timeline、video、audio 或 renderer 能力。
